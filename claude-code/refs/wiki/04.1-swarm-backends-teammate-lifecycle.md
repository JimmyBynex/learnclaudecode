# 4.1 Swarm Backends & Teammate Lifecycle

## Swarm Backends (Three Execution Environments)

| Backend | Implementation |
|---------|---------------|
| `InProcessBackend` | Runs agents as child processes |
| `TmuxBackend` | Spawns agents in tmux panes |
| `ITermBackend` | Creates iTerm2 tabs via AppleScript |

## Teammate Lifecycle (Three Phases)

1. **Initialization**: Generates unique task IDs with "t" prefix. Creates initial state with `startTime` and `pending` status.
2. **Execution & Layout**: `teammateLayoutManager` coordinates UI positioning through active terminal backends.
3. **Termination**: Process termination updates AppState to terminal statuses (completed, failed, killed).

## Permission Synchronization
`leaderPermissionBridge` mechanism: Leader maintains source-of-truth for permissions. Rule changes propagated to all active teammates. Global denial tracking system prevents repeated permission requests across swarm.

## Reconnection Logic
Handles terminal disconnects through task persistence in `AppState`. Reattachment capabilities for terminal multiplexer backends. Identification of terminal task states.

## TaskStateBase Fields

| Field | Purpose |
|-------|---------|
| Unique ID | "t" prefix identifier |
| Task type | Classification |
| Status | Current execution state |
| Description | Human-readable task summary |
| Output file path | Where output is stored |
| Start timestamp | Creation time |
| Output offset | Position tracking |
