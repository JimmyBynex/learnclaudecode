# 3.2 Shell & File Tools

## Core Tools

| Tool | Purpose |
|------|---------|
| `BashTool` | Shell command execution with multi-layered security pipeline (AST parsing, path verification, sed-specific validation) |
| `FileReadTool` | File contents with line-range support, `FileStateCache` for disk state, prevents context overflow on large files |
| `FileEditTool` | Targeted modifications via search/replace blocks or sed operations, optional structured diff UI review before finalization |
| `FileWriteTool` | Create new files or completely rewrite existing ones |
| `GlobTool` | File discovery via pattern matching, respects `.gitignore` rules |
| `GrepTool` | High-performance text search across repository, truncates large result sets with summaries |

## Security Framework

All file tools implement:
- Path normalization to resolve symlinks
- Block directory traversal
- `DenialTrackingState` tracks user-denied actions to prevent repeated failed attempts

Bash execution pipeline: bash classification → path validation → sed validation → execution.

## Execution Context (`ToolUseContext`)
- File state caching
- Abort controls for long operations
- Custom UI rendering capabilities
- Application state access (cost tracking)

## Permission Modes

| Mode | Behavior |
|------|---------|
| default | Prompts for destructive actions |
| bypass | Skips confirmations for trusted environments |
