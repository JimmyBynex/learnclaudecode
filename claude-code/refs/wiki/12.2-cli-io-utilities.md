# 12.2 CLI I/O Utilities

## Overview
Foundational I/O layer for structured communication between Claude Code and interfaces (terminal, JSON-RPC SDK hosts, remote execution environments).

## Key Components

### StructuredIO
Primary abstraction: converts raw input streams into `StdinMessage` or `SDKMessage` objects.

- **Request Management**: `pendingRequests` Map for async control requests, resolved on matching `control_response` receipt
- **Duplicate Prevention**: Tracks up to 1000 resolved `tool_use_id`s (LRU-style Set) to prevent Anthropic API 400 errors
- **Message Normalization**: `normalizeControlMessageKeys` for SDK protocol version compatibility
- **Permission Handling**: Generates `RequiresActionDetails` and executes permission request hooks

### RemoteIO
Extends StructuredIO for bidirectional streaming over network transports (SSE/WebSockets). Supports Remote Control and Cloud Code Runner (CCR) environments.

- Dynamic selection between SSETransport and WebSocketTransport
- Session ingress token injection and environment runner version headers
- Silent keep-alive frames every 30 seconds (prevents proxy timeouts)

**CCR v2 Integration:**
- State reporting of session metadata changes
- Transcript persistence via `writeInternalEvent`
- Command lifecycle tracking for reliable delivery

## Terminal Output Utilities

**print.ts**: Terminal rendering utilities bridging internal state to user terminal, integrating with Ink-based UI system.

**exit.ts Functions:**
- `cliError(msg)`: Writes to stderr, exits with code 1
- `cliOk(msg)`: Writes to stdout, exits with code 0

## NDJSON Serialization
`ndjsonSafeStringify.ts`: Safely serializes objects for NDJSON streams by escaping U+2028 (Line Separator) and U+2029 (Paragraph Separator) characters. Single-pass regex approach.
