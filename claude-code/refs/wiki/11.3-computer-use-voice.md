# 11.3 Computer Use & Voice

## Computer Use Architecture

Allows the agent to interact with the host operating system's GUI through clicking, typing, and screenshot capture. Provider-consumer pattern: LLM issues high-level commands → translated into OS-level events.

### Primary Components

| Component | Location | Function |
|-----------|----------|----------|
| Executor | `src/utils/computerUse/executor.ts` | Manages execution of computer use tool calls |
| Host Adapter | `src/utils/computerUse/hostAdapter.ts` | OS-level interface (macOS/Windows/Linux) |
| Swift Loader | `src/utils/computerUse/swiftLoader.ts` | Native Swift binary management for macOS |
| MCP Server | `src/utils/computerUse/mcpServer.ts` | MCP tool exposure for computer use |
| Gates | `src/utils/computerUse/gates.ts` | Safety checks and confirmation dialogs |

### Safety
Governed by strict safety gates providing context awareness and requiring explicit consent for dangerous actions. Permission dialogs in `src/components/PermissionDialogs.tsx`.

## Voice Integration (STT)

### Subsystem Components

| Component | Function |
|-----------|---------|
| `src/services/voice.ts` | Session lifecycle and audio hardware management |
| `src/voice/voiceStreamSTT.ts` | Real-time streaming and PCM audio chunking |
| `src/hooks/useVoice.ts` | React-based UI state management for microphone status |

### Voice Processing Pipeline
Audio capture → Streaming to transcription service → Text integration into CLI input buffer

### Key Operations
- `startListening()`: Initializes audio stream and STT session
- `stopListening()`: Closes stream and flushes remaining audio
- `processAudioChunk()`: Encodes raw audio for STT API compatibility

## Slash Commands
- `/voice`: Toggles voice input mode
- `/desktop`: Enables computer use tools
- `/chrome`: Browser-focused computer use mode
