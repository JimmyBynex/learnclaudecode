# 11.2 ULTRAPLAN & KAIROS

## ULTRAPLAN: Deep Planning Architecture

Offloads complex engineering tasks to remote high-capacity models (Opus 4.6). Enables CCR (Complex Capability Request) sessions for deep planning up to 30 minutes.

**Key Components:**
- **Keyword Detection**: Scans user input and agent thoughts for terms indicating architectural shifts, large-scale refactors, or complex debugging
- **CCR Session**: Dedicated remote execution context where Opus 4.6 generates multi-step plans
- **Remote Initiation**: Provisioned via Bridge API with access to current project state

**Execution Flow**: Local CLI sessions transition into ULTRAPLAN phases through QueryEngine, which communicates with Bridge API to establish remote planning environments.

---

## KAIROS: Proactive Monitoring System

KAIROS (Kinetic AI Real-time Operating System) enables Claude Code to operate proactively rather than reactively by monitoring system signals for improvement opportunities.

**Proactive Capabilities:**
- Log watching for stack traces and regressions
- Autonomous correction of failing builds/tests without user commands
- Background maintenance tasks during user idle periods

**System Components:**
- `KAIROS_MONITOR`: Core polling loop for system signals
- `ProactiveAction`: Specialized task type bypassing standard `ask()` loops
- `SignalClassifier`: Evaluates whether detected events warrant agent intervention

---

## Remote Opus 4.6 Integration

Bridge system handles remote session management:

1. **Provisioning**: `registerBridgeEnvironment` establishes remote workspaces
2. **Context Upload**: File snippets and `MEMORY.md` synchronize to ensure full project context
3. **Handover**: Local QueryEngine pauses; control transfers to `RemoteAgentTask`

**Auto Mode**: ULTRAPLAN/KAIROS operations use classifiers to determine if proactive actions require confirmation, applying safety guardrails through `getDefaultExternalAutoModeRules`.
