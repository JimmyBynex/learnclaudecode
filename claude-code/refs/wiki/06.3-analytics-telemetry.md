# 6.3 Analytics, Telemetry & Feature Flags

## Overview
Observability infrastructure covering: analytics event tracking (GrowthBook + Datadog), performance telemetry (OpenTelemetry → BigQuery), and remote feature configuration (A/B testing + emergency deactivation).

## State & Provider Initialization
Global telemetry state in the `STATE` singleton:

| Component | Purpose |
|-----------|---------|
| `Meter` | Creates OpenTelemetry metrics |
| `LoggerProvider` | Manages structured event output |
| `TracerProvider` | Handles request and tool execution tracing |
| `SessionId` | Unique identifier per CLI process |
| `EventLogger` | OTel-based structured event system |

## Metrics Tracking
- Cost tracking (USD per model)
- Token consumption (input/output)
- Agent processing duration
- File edit success/failure rates
- Session lifecycle events

## Analytics Architecture

### GrowthBook Integration
Remote configuration enables:
- Selective tool enable/disable
- Rate limit threshold adjustments
- Experimental UI component rollout
- Local or inline flag configuration

### Event Flow
Events pass through `firstPartyEventLogger` → opt-out check → anonymize identifiers → batch transmissions.

## OpenTelemetry Instrumentation

Three signal types:
- **Spans**: LLM request and tool execution timing
- **Logs**: Structured entries with trace IDs
- **Metrics**: Aggregated cost and latency percentiles

### BigQuery Export
Telemetry transforms into SQL-queryable schema for session reconstruction and analysis.

### Session Lineage
`sessionId` + `parentSessionId` relationship tracks task hierarchies when agents spawn sub-tasks.

## Killswitch Pattern (multi-tier data collection halt)

1. **User Config**: Local `analytics: false` setting
2. **Environment Override**: `CLAUDE_CODE_NO_TELEMETRY` variable
3. **Remote Toggle**: GrowthBook flag for global sink deactivation

When disabled, events drop immediately. When enabled, enrichment adds `clientType` and `sessionId` before sink transmission.
