# 13.1 Settings Architecture

## Overview
Centralized settings management for user preferences, machine-specific configurations, and MDM (Mobile Device Management) policies. Uses in-memory cache with change detection.

## Core Components

| Component | Function |
|-----------|---------|
| `settings.ts` | Primary API for reading/writing settings |
| `settingsCache.ts` | In-memory cache reducing disk I/O |
| `validation.ts` | Schema validation for settings objects |
| `applySettingsChange.ts` | Merges updates, triggers side effects |
| `types.ts` | TypeScript definitions for Settings |
| `constants.ts` | Default values and filesystem paths |

## Settings Schema

Key fields:
- `model`: Preferred Claude model alias (e.g., `sonnet-20241022`)
- `theme`: UI styling preferences
- `permissions`: Default tool execution behavior (`not-set`, `always-allow`)
- `pluginOnlyPolicy`: Restricts agent to approved plugins only

## Data Flow & Precedence (3-tier hierarchy)

1. **MDM/Managed Settings** (highest priority): Non-overridable organizational policies
2. **User Settings**: Local JSON configuration file values
3. **Defaults** (lowest priority): Hardcoded fallback values

`allowedSettingSources` in global state enforces which sources are valid during a session.

## Change Detection Pattern
Avoids polling via `changeDetector` mechanism:
- `applySettingsChange` compares new vs cached state
- Differences trigger subscriber notifications (React hooks, `AppStateStore`)
- Manual cache clearing: `resetSettingsCache`

## MDM & Enterprise Features
- `rawRead`: Internal utility bypassing cache to access source-of-truth policy files
- `pluginOnlyPolicy`: Restricts agents to plugin-approved tools only when enabled
- `SettingSource`: Enum tracking setting origin (MDM, user, flag) for security compliance

## Validation & UI Integration
- `/config` command: visual management interface rendering Settings React component
- `validation.ts`: Prevents invalid data (non-existent models, invalid permission modes)
- Complex types like `PluginHookMatcher` also validated
