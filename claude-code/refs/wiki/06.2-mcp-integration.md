# 6.2 MCP (Model Context Protocol) Integration

## Overview
Foundational service enabling Claude Code's AI agent to connect with external tools, resources, and prompts from local or remote servers through a standardized interface.

## Core Architecture

**Connection Management**

| Component | Function |
|-----------|---------|
| `MCPConnectionManager` | Manages active `McpClient` instances and lifecycles |
| `connectToServer` | Factory function instantiating correct transport (stdio/SSE) based on config |
| `McpClient` | Wraps underlying MCP SDK client with standardized error handling |
| `getMcpConfigByName` | Retrieves server config from project, user, or global scopes |
| `getMcpServerConnectionBatchSize` | Determines concurrency limits for health checks |

## Transport Mechanisms

Three primary transport types:
1. **Stdio** — Spawns local process communicating via stdin/stdout
2. **SSE (Server-Sent Events)** — Remote HTTP endpoint with persistent duplex communication
3. **VS Code SDK** — Specialized transport for VS Code extension host interaction

## Security Framework

| Component | Purpose |
|-----------|---------|
| `channelAllowlist.ts` | Defines trusted servers bypassing interactive prompts |
| `channelPermissions.ts` | Restricts which MCP servers can perform specific actions |
| `elicitationHandler.ts` | Manages UI flow when servers request user input |

## CLI Integration

| Handler | Function |
|---------|---------|
| `mcpListHandler` | Checks server health concurrently, reports connection status |
| `mcpAddHandler` | Registers new MCP servers with config validation |
| `mcpRemoveHandler` | Uninstalls servers, cleans secure storage tokens |
| `mcpServeHandler` | Starts local MCP server instances for debugging |

## Authentication
- `xaaIdpLogin`: Handles OAuth/OIDC flows for Identity Providers
- `officialRegistry.ts`: Maintains verified Anthropic MCP server list
- Secure token storage via platform keychain with cleanup on server removal
