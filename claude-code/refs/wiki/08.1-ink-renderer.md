# 8.1 Ink Renderer & Terminal Engine

## Overview
Claude Code's terminal UI uses a custom Ink implementation with specialized reconciler, layout engine, and terminal I/O layer optimized for AI agent interactions and streaming text.

*Note: This is the React/Ink-based UI layer. In a Go rewrite, this would be replaced by bubbletea/lipgloss.*

## System Architecture

Unidirectional data flow: React components → custom DOM-like tree → Yoga layout engine → ANSI-encoded terminal output.

## Core Components

### 1. Custom Reconciler & DOM Model (`src/ink/dom.ts`)
- Internal terminal UI representation optimized for fixed-width grids
- Translates React elements into tree supporting terminal-specific attributes (color, backgroundColor, dim)

### 2. Layout Engine (Yoga)
- `yoga-layout` for Flexbox-like functionality in terminal environments
- Enables complex responsive designs (side-by-side agent views, auto-wrapping text)
- Maps `DOMNode` properties to Yoga flexbox properties

### 3. Terminal Engine & I/O

| Feature | Implementation |
|---------|----------------|
| Raw Mode | `setupRawMode()` captures individual keystrokes |
| Bidi Text | Bidirectional rendering for RTL languages |
| Selection/Scrolling | Manages scrollback buffer and text selection |
| Cursor Management | Precise positioning during streaming LLM responses |

## Frame & Output Pipeline
- Calculates differences between current and previous frames → minimize ANSI escape sequences
- Critical for performance during high-speed token streaming

### Performance Optimization
1. **Partial Updates**: Only changed lines re-rendered
2. **Batching**: Multiple state updates consolidated into single frame render
3. **Yoga Caching**: Layout calculations cached unless node tree or dimensions change

## Integration with App State
Ink renderer couples with `AppStateStore`. When `QueryEngine` emits tokens or tools execute, state updates trigger React re-renders. Ink reconciler calculates minimal UI updates for terminal display.
