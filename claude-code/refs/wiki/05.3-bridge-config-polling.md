# 5.3 Bridge Configuration & Polling

## Core Configuration & Entitlements

### Bridge Feature Gating
- **Build Flag**: `BRIDGE_MODE` enabled at build time
- **Subscription**: `authModule.isClaudeAISubscriber()`
- **GrowthBook Gates**: `tengu_ccr_bridge` gate for user's organization
- **V2 Path**: `tengu_bridge_repl_v2`

### Version Requirements
- V1 (Env-based): `tengu_bridge_min_version`
- V2 (Env-less): `min_version` from `tengu_bridge_repl_v2_config`

### Connection URLs
- `getBridgeBaseUrl()`: API base, prioritizing `CLAUDE_BRIDGE_BASE_URL` for internal dev
- `buildBridgeSessionUrl()`: Constructs user-facing URLs with `?bridge={environmentId}`

## Polling & Liveness

### Poll Interval Configuration

| State | Value | Purpose |
|-------|-------|---------|
| not_at_capacity | 2000ms | Seeking new remote sessions |
| at_capacity | 600,000ms | Maintaining active connection liveness |
| non_exclusive_heartbeat_interval_ms | 0 | Disabled by default |
| reclaim_older_than_ms | 5000ms | Threshold for unacknowledged work |

### CapacityWake Utility
Coordinated sleeping during "at capacity" states:
- `createCapacityWake(outerSignal)`: Returns controller merging AbortSignal + manual `wake()` trigger
- `wake()`: Aborts current sleep to re-check for work

## Session Persistence & Recovery

### Bridge Pointers
`BridgePointer`: `{sessionId, environmentId, source}` stored as `bridge-pointer.json`.
- **TTL**: 4-hour staleness threshold (`BRIDGE_POINTER_TTL_MS`)
- **Worktree Awareness**: `readBridgePointerAcrossWorktrees` scans up to 50 siblings
- **Cleanup**: On clean shutdown or stale detection

## Bridge API Client (CCR v2)

| Function | Endpoint | Purpose |
|----------|---------|---------|
| `createCodeSession()` | POST `/v1/code/sessions` | Initialize remote environment |
| `fetchRemoteCredentials()` | POST `/v1/code/sessions/{id}/bridge` | Get `worker_jwt` and `api_base_url` |

**RemoteCredentials**: JWT, expiry, `worker_epoch` (increments on each registration).

## BridgeLogger & Status UI

State machine tracking visual bridge connection state:
- **Status States**: idle, attached, titled, reconnecting, failed
- **QR Display**: Generates/toggles QR codes for mobile connection
- `countVisualLines()`: Tracks terminal rows for clean clearing
- `computeShimmerSegments()`: Splits strings for shimmer animation with grapheme segmentation

## Debugging & Fault Injection
- `wrapApiForFaultInjection()`: Wraps BridgeApiClient to intercept calls + throw errors from `faultQueue`
- `injectBridgeFault()`: Queues specific failures to test recovery machinery
