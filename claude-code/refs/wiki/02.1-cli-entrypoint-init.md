# 2.1 CLI Entrypoint & Initialization

## Overview
Claude Code (Tengu) follows a three-phase startup sequence:

1. **Argument Parsing** via Commander.js in `src/main.tsx`
2. **Bootstrap & Initialization** through `src/entrypoints/init.ts`
3. **REPL Launch** into React/Ink terminal interface

## Key Initialization Steps

| Step | Details |
|------|---------|
| Keychain Prefetch | Retrieves OAuth tokens + API keys from OS keychain via `src/utils/secureStorage/` |
| MDM & Managed Settings | Loads organization-level policy configurations |
| Telemetry Bootstrap | Initializes OpenTelemetry + BigQuery export for session tracking |
| MDM Remote Fetch | Retrieves feature restrictions from enterprise policies |

## Global CLI Options
- `--verbose`: Debug logging
- `--dangerously-skip-permissions`: Bypass permission checks
- `--model`: Override default Claude model

## React & Ink Integration
CLI uses Ink for terminal rendering (React components + yoga-layout for positioning), not direct stdout writes.

## REPL Launcher Responsibilities
- Creates singleton `STATE` object (cost + token tracking)
- Registers slash commands from `src/commands.ts`
- Establishes input pipeline for processing user requests

## Exit Handling
- `cliError()`: Prints to stderr, exits with code 1
- `cliOk()`: Prints to stdout, exits with code 0
- `update()`: Handles version checking and installation type detection
