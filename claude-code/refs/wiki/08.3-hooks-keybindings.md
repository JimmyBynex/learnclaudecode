# 8.3 Hooks & Keybindings

*Note: React-specific hooks layer. In Go, equivalent state management would be handled differently.*

## React Hooks Architecture
Bridges global state, external services (MCP, Bridge API), and terminal UI. Key hooks in `src/hooks/`, `src/ink/hooks/`, `src/buddy/`.

## Tool & Permission Hooks
- `useCanUseTool`: Evaluates tool execution authorization based on current `PermissionMode` and rules
- `useSettings`: Provides user configuration access and global settings state updates

## Swarm & Task Management
- `useSwarmInitialization`: Configures swarm backend (InProcess, Tmux, iTerm), establishes communication channels
- `useTasksV2`: Reactive task registry views for progress indicators and concurrent agent activity

## Remote & Bridge Integration
- `useReplBridge`: Connects local terminal UI to `replBridge` transport, manages inbound messages and attachment sync

## Voice & Companion
- `useVoice`: Manages Voice-to-Text state (recording status, transcript handling)
- `useBuddyNotification`: UI notifications based on companion stats (DEBUGGING, CHAOS, SNARK) or evolution events

## Keybindings System
Maps keyboard inputs to application actions. Supports custom configurations including Vim-style navigation via `KeybindingProvider`.

- **Vim Mode**: hjkl navigation + modal editing via `/vim` command
- **Custom Bindings**: Users remap actions via `/keybindings` interface
- **Global Shortcuts**: Agent interruption (Ctrl+C), prompt submission

## Ink Integration Hooks (`src/ink/hooks/`)
- `useInput`: Integrates Ink input handling with keybinding resolver
- `useFocus`: Manages focus across terminal UI components
- Terminal dimension hooks: track resize events for layout re-triggering

## Subcommand Handler Integration
Handlers like `setup-token` and `doctor` wrap UI logic in `AppStateProvider` + `KeybindingSetup` providers, ensuring consistent behavior outside main REPL loop.
