# 8.2 React Components & Design System

*Note: This is the React/Ink UI layer. In a Go rewrite, the equivalent would be bubbletea components.*

## Component Architecture
- Unidirectional data flow driven by global STATE object
- Components in `src/components/`, terminal primitives in `src/ink/`

## Primary UI Components

### App.tsx
Root component orchestrating CLI interface. Manages layout: message history, persistent prompt input, permission dialogs. Uses `Box` and `Transform` from Ink for terminal resizing/layout constraints.

### Messages Component
Renders conversation history:
- Human messages (user text + slash commands)
- AI messages (responses, thought blocks, tool calls)
- Tool results (bash output, file reads, LSP queries)

### PromptInput
- Real-time syntax highlighting for commands
- File/command autocomplete with MCP tool suggestions
- Multi-line editing via Shift+Enter
- History navigation with arrow keys

### PermissionDialogs
Intercepts dangerous/sensitive tool requests. Supports YOLO (auto-approve), Default (ask), Bypass (admin) modes.

## Specialized Components

| Component | Purpose |
|-----------|---------|
| `StructuredDiff` | Color-coded file change display for agent proposals |
| `AgentProgressLine` | Real-time thinking/execution status with spinner and elapsed time |
| `CompanionSprite` | BUDDY animated companion: 500ms animation cycles, dynamic speech bubbles, Mulberry32 PRNG seeded by userId |

## Design System

**Color & Theme**: Semantic palette mapping logical names to ANSI codes

**Layout Primitives**: Box (flexbox), Text (bold/italic/dim), Spinner, LogoV2 (ASCII art)

**Responsive Constraints**: Full sprites render only when terminal width > 100 columns
