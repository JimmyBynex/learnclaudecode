# 11.1 BUDDY Companion System

## Overview
Persistent, deterministic digital companion alongside the user's terminal input. Includes gacha-style collection mechanics, evolution tracking, and LLM integration.

## Core Data Structures
- **CompanionBones**: Immutable physical traits (species, eye, hat, rarity, shiny, base stats)
- **CompanionSoul**: Mutable identity traits (name, personality)
- **Companion**: Complete object = Bones + Soul + hatchedAt timestamp
- **StoredCompanion**: Persisted subset in global configuration

## Generation System
Deterministic Mulberry32 PRNG seeded by hashing user's ID with static salt (prevents save-scumming).

Generation flow:
1. User identification via `accountUuid` or `userID`
2. ID hashing using `Bun.hash` when available
3. Rarity determination via weighted probabilities

## Rendering and Animation

### Sprite System
- Sprites: 5 lines tall, 12 characters wide
- Eye types rendered via placeholder substitution in body templates
- Idle sequence: rest, fidget, rare blink frames
- Petting mechanic: heart frames for `PET_BURST_MS` duration

### Display Management
- `companionReservedColumns`: calculates required horizontal space
- Sprite hidden when terminal width < `MIN_COLS_FOR_FULL_SPRITE` (100)
- Speech bubble component handles text wrapping and tail positioning

## Prompt Integration
Injects `companion_intro` attachment into message history via `getCompanionIntroAttachment`. LLM instructions:
- Small species is present with companion's name
- LLM is NOT the companion itself
- How to respond if user addresses companion directly
