# 11.4 Teleport & Remote Environments

## Overview
Teleport system enables moving Claude Code session context between environments (local → remote cloud, or different project directories). Manages environment discovery, state packaging via Git bundles, and remote session resumption orchestration.

## Core Components

| Component | Path | Function |
|-----------|------|----------|
| Teleport API | `src/utils/teleport/api.ts` | Backend communication for registration and event tracking |
| Environment Selection | `src/utils/teleport/environmentSelection.ts` | Queries available remote environments, prompts user |
| Git Bundling | `src/utils/teleport/gitBundle.ts` | Creates compressed repository state bundles for transfer |
| Environment Definitions | `src/utils/teleport/environments.ts` | Type definitions and metadata for supported remote environments |
| UI Components | `src/components/TeleportProgress.tsx` | Step-by-step progress display |

## Functional Processes

**Git Bundling Workflow:**
- Identifies uncommitted changes and staged files
- Uses `git bundle create` to generate binary delta representation
- Packages for transmission to remote initialization endpoint

**Teleportation UI:**
- `TeleportProgress`: Step-by-step display (bundling → uploading → starting remote session)
- `TeleportError`: Failure state management with recovery guidance

**Cross-Project Resume:**
Serializes `AppState` from `src/state/AppStateStore.ts` to retain agent memory while switching working directories.

## Bridge API Integration
1. **Handshake**: `teleport/api.ts` calls `registerBridgeEnvironment` to prepare target environments
2. **State Transfer**: Git bundles uploaded via `filesApi` in `src/services/api/client.ts`
3. **Resumption**: Remote `replLauncher` triggered with session ID for conversation continuation

## Related Slash Commands
- `/teleport`: Migrate session state to remote environment
- `/remote-env`: Manage remote environments
- `/remote-setup`: Provision remote environment
