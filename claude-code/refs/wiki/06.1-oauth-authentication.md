# 6.1 OAuth & Authentication System

## Overview
Claude Code implements OAuth 2.0 with PKCE flow to manage user identity and API authorization. Interfaces with Anthropic's identity providers (Console and Claude.ai). Handles secure credential storage across operating systems.

## Core Architecture

**OAuthService** — primary coordination class managing OAuth 2.0 PKCE flow.

Supports two login methods:
- **Console-based**: API key generation
- **Claude.ai-based**: Native web sessions

### Key Functions

| Function | Purpose |
|----------|---------|
| `installOAuthTokens` | Post-acquisition logic: clears old state via `performLogout`, fetches user profile, stores account metadata |
| `refreshOAuthToken` | Background rotation of expired access tokens using stored refresh tokens |
| `getOauthProfileFromOauthToken` | Retrieves account info (email, name, organization) |
| `storeOAuthAccountInfo` | Persists account details to local configuration |

## Secure Storage Strategy

Tiered credential storage:
1. **macOS Keychain**: Primary storage using `security` CLI
2. **Generic Keychain**: `keytar` or native bindings
3. **Fallback Storage**: Local encrypted/obfuscated files when keychain unavailable

## Authentication Flow (`authLogin` handler)

Four modes:
1. Standard flow — opens browser with local redirect listener
2. Environment refresh — via `CLAUDE_CODE_OAUTH_REFRESH_TOKEN`
3. SSO/Enterprise — forced login methods
4. Token installation — profile fetching + API key creation

## Security Implementation
PKCE cryptography layer generates secure `code_verifier` and `code_challenge` strings. Even if authorization code is intercepted, it cannot be exchanged for tokens without original verifier.
