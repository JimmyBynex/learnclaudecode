# 5.1 Bridge API Client & Authentication

## Overview
The Bridge API Client serves as the primary communication layer between the local Claude Code CLI and the Anthropic Bridge service, enabling remote control sessions where external IDEs or web interfaces can operate the CLI as a worker.

## Core Architecture
Long-polling and heartbeat pattern: CLI registers as environment → polling loop for work → heartbeat signals to maintain availability.

## Authentication Layers

### Layer 1: OAuth with 401 Retry
`withOAuthRetry` manages API calls, handles expired tokens by triggering refresh via `onAuth401` before retrying.

### Layer 2: Trusted Device Tokens (CCR v2)
Requires `X-Trusted-Device-Token` header for elevated security sessions.
- **Enrollment**: `enrollTrustedDevice()` → `POST /api/auth/trusted_devices`
- **Storage**: system keychain via `getSecureStorage()`
- **Control**: `tengu_sessions_elevated_auth_enforcement` feature flag

### Layer 3: JWT Session Tokens
Short-lived JWT tokens (Session Ingress Tokens). `createTokenRefreshScheduler` proactively refreshes with 5-minute buffer before expiry.

## Key Functions

| Function | Purpose |
|----------|---------|
| `registerBridgeEnvironment` | Transmits machine name, directory, max sessions; supports idempotent re-registration via `reuseEnvironmentId` |
| `pollForWork` | Long-polls `/v1/environments/{id}/work` until work available or timeout |
| `heartbeatWork` | Signals active session processing |
| `acknowledgeWork` | Confirms work receipt and worker responsibility |

## Error Classification

| Status | Error Type | Behavior |
|--------|-----------|----------|
| 401 | Unauthorized | OAuth retry or fatal error |
| 403 | Forbidden | Fatal error (environment expired) |
| 404 | Not Found | Fatal error (environment not found) |

## Session Utilities

- **Work Secrets**: Base64url-encoded JSON objects containing session ingress tokens and API base URLs (validated by version "1")
- **Session ID Normalization**: `sameSessionId` compares UUID bodies across different infrastructure layer prefixes (CCR v2 vs Legacy)
- **Worker Registration (CCR v2)**: Bridge registers as worker to obtain `worker_epoch` for heartbeat/event request consistency
- **JWT Helpers**: `decodeJwtPayload` strips `sk-ant-si-` prefix; `decodeJwtExpiry` extracts `exp` claim as Unix seconds
